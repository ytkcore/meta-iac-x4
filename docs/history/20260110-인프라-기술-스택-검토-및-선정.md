# 20260110-인프라-기술-스택-검토-및-선정

## 1. 개요
프로젝트 착수 전, 요구사항(보완성, 비용 효율성, 관리 편의성)을 충족하기 위한 최적의 인프라 기술 스택을 검토하고 확정함.

## 2. 주요 작업 내용
- **K8s 배포판 선정**: EKS(Managed) 대신 비용 및 커스텀 유연성이 높은 **RKE2 (Self-managed)**를 최종 선정.
- **IaC 도구 확정**: 상태 관리 및 모듈화 가독성이 높은 **Terraform**을 표준 도구로 채택.
- **보안 전략 수립**: AWS ACM을 통한 TLS 종료 및 Security Group 기반의 계층형 방어 모델 초안 수립.

## 3. 결과 및 가치
- 프로젝트의 기술적 방향성을 명확히 하여 아키텍처 일관성 확보.
- 향후 확장성을 고려한 벤더 종속성(Lock-in) 최소화 전략 수립.
