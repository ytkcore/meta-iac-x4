# 국내 접근제어 트렌드 및 현황

> 한국 시장의 특수성과 규제 환경을 반영한 접근제어 전략

---

## 1. 한국 시장의 특수성

### 규제 환경

| 분야 | 규제 | 영향 |
|:---|:---|:---|
| **금융** | 전자금융감독규정, 망분리 의무 | 물리적/논리적 망분리 필수 |
| **공공** | 국정원 보안 가이드라인 | CC 인증 제품 우선 적용 |
| **의료** | 개인정보보호법, 의료법 | 민감정보 접근 로그 5년 보관 |
| **전체** | ISMS-P | 연 1회 인증 심사, 접근통제 증적 필수 |

### 국내 기업 Pain Points

```
❌ "Cloudflare? 해외 서버라 규제 이슈 있어요"
❌ "SaaS는 심사 때 설명하기 어려워요"
❌ "오픈소스 쓰면 보안팀에서 검증 요청해요"
❌ "기존 VPN 인프라가 있어서 바꾸기 어려워요"
```

## 2. 국내 기업별 도입 현황

### 금융권 (토스, 카카오뱅크, 뱅크샐러드)

| 솔루션 | 도입 현황 | 특이사항 |
|:---|:---|:---|
| **Teleport** | ✅ 높음 | 세션 녹화, ISMS-P 감사 대응 용이 |
| **HashiCorp Vault** | ✅ 높음 | Dynamic Secrets, 금감원 권고 수준 충족 |
| **Tailscale** | 🔶 개발망만 | 프로덕션은 자체 VPN 유지 |
| **Cloudflare** | 🔶 제한적 | WAF 위주, Access는 규제 검토 중 |

**토스 사례:**
- 내부적으로 **Teleport + Vault** 조합 운영
- 모든 인프라 접근에 MFA + Just-in-Time 승인
- 세션 녹화로 ISMS-P 심사 시 증적 자동 제출

---

### 커머스/플랫폼 (쿠팡, 배민, 당근)

| 솔루션 | 도입 현황 | 특이사항 |
|:---|:---|:---|
| **AWS Client VPN** | ✅ 높음 | 기존 인프라 연속성 |
| **Tailscale** | ✅ 확산 중 | 개발 생산성 향상 |
| **Cloudflare Tunnel** | ✅ 확산 중 | 내부 도구 노출용 |
| **자체 PAM** | 🔶 대기업만 | 삼성SDS, LG CNS 등 |

---

### 테크 스타트업 (스몰캡)

| 솔루션 | 도입 현황 | 이유 |
|:---|:---|:---|
| **Tailscale** | ✅ 매우 높음 | 무료, 5분 설치 |
| **Cloudflare Access** | ✅ 높음 | 무료 티어, SSO 통합 |
| **Teleport Cloud** | 🔶 보통 | 유료지만 운영 부담 제로 |
| **자체 VPN** | ❌ 낮음 | 운영 인력 없음 |

## 3. 국산 PAM/VPN 솔루션

| 제품 | 벤더 | 특징 | 가격대 |
|:---|:---|:---|:---:|
| **SecuID** | 펜타시큐리티 | 특권계정관리, CC 인증 | 고가 |
| **NetHelper** | 넷헬퍼 | 망분리, 파일전송 통제 | 중가 |
| **SecureGate** | 지란지교 | 접근통제, 세션 녹화 | 중가 |
| **HIWARE** | 에스에스앤씨 | 서버 접근관리, DB 접근관리 | 중가 |
| **DBSafer** | 피앤피시큐어 | DB 접근통제, 쿼리 감사 | 중가 |

### 장점
- CC 인증, ISMS-P 심사 시 유리
- 한국어 지원, 국내 기술 지원
- 규제 기관과의 커뮤니케이션 용이

### 단점
- 멀티클라우드/온프레미스 통합 어려움
- 모던 인프라(K8s) 지원 부족
- UI/UX가 레거시 수준

## 4. 글로벌 vs 국산 비교

| 기준 | 글로벌 (Tailscale, Cloudflare) | 국산 (SecuID, HIWARE) |
|:---|:---|:---|
| **K8s 네이티브** | ★★★★★ | ★★☆☆☆ |
| **멀티클라우드** | ★★★★★ | ★★☆☆☆ |
| **ISMS-P 대응** | ★★★☆☆ (문서 작업 필요) | ★★★★★ (기본 제공) |
| **비용** | 무료~저가 | 중~고가 (라이선스) |
| **구축 속도** | 1시간 | 1~2주 |
| **운영 부담** | 낮음 (SaaS) | 높음 (On-prem) |
| **벤더 종속** | 낮음 | 높음 |

## 5. ISMS-P 대응 포인트

### 접근통제 감사 항목 (ISMS-P 2.5)

| 항목 | Teleport | Cloudflare | Tailscale | 국산PAM |
|:---|:---:|:---:|:---:|:---:|
| 접근 로그 | ✅ | ✅ | ✅ | ✅ |
| 세션 녹화 | ✅ | ❌ | ❌ | ✅ |
| 명령어 감사 | ✅ | ❌ | ❌ | ✅ |
| 승인 워크플로우 | ✅ | ✅ | ✅ | ✅ |
| 한글 보고서 | 🔶 | ❌ | ❌ | ✅ |

### 실무 Tip
- **세션 녹화**가 필요한 구간만 Teleport 적용
- 나머지는 Tailscale/Cloudflare로 비용 절감
- ISMS-P 심사 시 "글로벌 표준 + 보완 통제" 논리로 설명

## 6. 시나리오별 권장안

| 환경 | 권장 조합 | 이유 |
|:---|:---|:---|
| **스타트업 (시드~A)** | Tailscale + Cloudflare (무료) | 비용 최소화, 빠른 구축 |
| **스케일업 (B~C)** | Teleport + Tailscale | 감사 대응 준비 |
| **금융/공공** | Teleport + 국산 PAM | 규제 대응 필수 |
| **대기업** | HashiCorp (Boundary+Vault) + 국산 PAM | 기존 인프라 연동 |

## 7. 구현 로드맵

### Phase 1: 즉시 (1일)
- Tailscale 도입: Bastion/Node에 설치
- Public LB 제거: Service를 ClusterIP로 변경

### Phase 2: 단기 (1주)
- Cloudflare Tunnel 구축: Web UI들을 Tunnel 뒤로 이동
- SSO 정책 적용: `@company.com` 도메인만 허용

### Phase 3: 중기 (1개월)
- Teleport 도입: SSH/K8s 세션 녹화, 감사 로그
- Device Posture: OS 업데이트, 디스크 암호화 검증

### Phase 4: 장기 (규제 대응 필요 시)
- HashiCorp Boundary + Vault: Dynamic Secrets
- 국산 PAM 병행: ISMS-P 심사 대응
